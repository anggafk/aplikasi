Urutan Pengerjaan yang Disarankan
▼
Kerjakan secara berurutan karena ada dependensi antar modul:

1. Basic Config → timezone, domain, password, IP VLAN (semua device dulu)
2. L2 Services → VTP → VLAN → STP → Trunk → EtherChannel → HSRP
3. OSPF (COR1) dan EIGRP (COR2) — bisa paralel
4. BGP → setelah OSPF & EIGRP konvergen
5. IP Services → NAT & DHCP setelah routing sudah jalan
6. Security & VPN → DMVPN + SSH + Port Security terakhir
⚠ Penting: Tidak boleh menggunakan static route kecuali di RG1/RG2 bagian Security & VPN. Pastikan semua interface no shutdown sebelum submit.

Gejala
Router PT2800 sering boot ke ROM Monitor (rommon) karena configuration register yang salah. Jika CLI hanya menampilkan rommon 1 >, rommon 2 >, dan seterusnya — router tidak bisa dikonfigurasi. Perbaiki ini SEBELUM konfigurasi apapun di RG1 & RG2.
! Gejala yang muncul di CLI:
!   rommon 1 >
!   rommon 2 >
!   (dst...)
Langkah perbaikan — ketik di rommon prompt
! RG1 DAN RG2 — lakukan di keduanya
rommon 1 > confreg 0x2102
rommon 2 > reset

! Router akan reboot otomatis.
! Tunggu hingga muncul prompt normal:
!   Router>
! Baru lanjutkan konfigurasi basic di bawah.
Verifikasi — setelah reboot
show version | include Configuration register
! Harus menampilkan: Configuration register is 0x2102

Router> enable
Router# show running-config
! Harus masuk IOS normal, bukan rommon
⚠ Jika masih rommon setelah reset:
1. Klik device RG1/RG2 di Packet Tracer → tab Physical → pastikan IOS image terpasang
2. Atau ganti device ke Router 2911 / 4321 yang lebih stabil di Packet Tracer 8.x

T3Trunk Port
S3COxS2CO1S2CO2SRG1SRG2
▼
S3COx — trunk manual ke S2COx (native VLAN 888)
Setelah S3COx dikonfigurasi trunk, DTP otomatis menegosiasikan sisi S2COx menjadi trunk — switchport mode trunk tidak perlu diketik di S2COx.
! S3CO1 & S3CO2 — port G0/3 ke S2CO1
interface g0/3
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk native vlan 888

! S3CO3 & S3CO4 — port G0/3 ke S2CO2
interface g0/3
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk native vlan 888
S2CO1 & S2CO2 — cukup samakan native VLAN 888 di kedua interface
S2COx punya 2 interface trunk — satu ke S3COx pertama, satu ke S3COx kedua. Keduanya harus disamakan native VLAN-nya agar tidak muncul pesan native VLAN mismatch. Mode trunk sudah otomatis lewat DTP.
! S2CO1 — G0/1 ke S3CO1, G0/2 ke S3CO2
interface range g0/1 - 2
 switchport trunk native vlan 888

! S2CO2 — G0/1 ke S3CO3, G0/2 ke S3CO4
interface range g0/1 - 2
 switchport trunk native vlan 888
SRG1 & SRG2 — trunk ke router RG1/RG2 (native VLAN 1, default)
SRG1/SRG2 tidak terhubung ke S3COx, melainkan ke router RG1/RG2. Interface G0/0 dikonfigurasi trunk agar VLAN 100-103 bisa dilewatkan ke subinterface router (G0/1.100, G0/1.101, dst). Native VLAN dibiarkan default (VLAN 1) sesuai soal.
! SRG1 — G0/0 terhubung ke RG1
interface g0/0
 switchport trunk encapsulation dot1q
 switchport mode trunk

! SRG2 — G0/0 terhubung ke RG2
interface g0/0
 switchport trunk encapsulation dot1q
 switchport mode trunk
Pengujian keseluruhan setelah T3
! Cek trunk aktif di S3COx:
show interfaces trunk
! G0/3 harus muncul, native VLAN = 888

! Di S2CO1 / S2CO2 — VLAN sudah propagasi dari VTP:
show vlan brief
! VLAN 100-103 harus sudah muncul

! Di SRG1 / SRG2 — cek trunk ke RGx:
show interfaces trunk
! G0/0 harus muncul sebagai trunk

T6Default Route CO1 → OSPF (Type 2, Metric 5000)
CO1RCO4
▼
CO1 advertise default route ke OSPF jika dapat dari BGP
! CO1
router ospf 100
 default-information originate metric 5000 metric-type 2
RCO4 inject default route ke OSPF dengan kondisi G0/3 harus up (metric 4000)
! RCO4 — conditional default route via track
ip sla 1
 icmp-echo 10.10.0.209
ip sla schedule 1 life forever start-time now

track 1 ip sla 1 reachability

ip route 0.0.0.0 0.0.0.0 null0 track 1

router ospf 100
 default-information originate metric 4000 metric-type 2
Catatan: Conditional default route RCO4 hanya aktif saat interface G0/3 (10.10.0.208/30) ada di routing table. IP SLA memonitor next-hop di segmen tersebut.

T6Port Security — CO1-CLI & CO2-CLI
S2CO1S2CO2
▼
Buat VLAN 150 VOIP, aktifkan port security max 2 MAC, auto-recovery 3 menit
! S2CO1 & S2CO2 — buat VLAN VOIP dulu
vlan 150
 name VOIP

! S2CO1 — port ke CO1-CLI
interface g0/0
 switchport mode access
 switchport access vlan 101
 switchport voice vlan 150
 switchport port-security maximum 2
 ! max 2 = 1 IP phone + 1 PC
 switchport port-security violation shutdown
 switchport port-security
 spanning-tree portfast

! Auto-recovery setelah 180 detik (3 menit):
errdisable recovery cause psecure-violation
errdisable recovery interval 180

! S2CO2 — port ke CO2-CLI, konfigurasi sama
Pengujian
show port-security interface g0/0
! Port Security: Enabled
! Maximum MAC Addresses: 2
! Violation Mode: Shutdown

show errdisable recovery
! psecure-violation: Enabled, interval: 180 sec
